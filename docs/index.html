How It Works

The 7-Step Autonomous Generation Engine.

A coordinated sequence across intelligence, sandbox analysis, AI generation, and live validation.

1

String Selection

Raw binary strings are scored and filtered using a weighted pattern library. High-signal strings (URLs, APIs, crypto hints, PS variables) are prioritized. Cap: 80 for PE / 120 for scripts.

Scoring GateDeduplication
2

Context Building

PE metadata (imphash, sections, imports), CAPA behavioral capabilities, MITRE ATT&CK techniques, and VT threat labels are assembled into a structured JSON payload.

CAPApefileVirusTotal
3

AI Generation

A persistent Gemini 2.5 Flash chat session receives the full context payload. Temperature 0.2. The AI generates a YARA rule following strict path-specific CRITICAL REQUIREMENTS.

Gemini 2.5 FlashStateful Chat
4

Normalization

Rule text is cleaned: markdown fences stripped, ascii wide enforced on text strings, import "pe" auto-injected if pe.* is used, duplicate imports deduplicated.

Auto-Fix
5

Policy Gate

Deterministic checks: string count minimums (7 PE / 15 non-PE), dual PE anchors, N−2 condition threshold, regex ban, entropy field ban, import statement validation.

Policy Enforcement
6

Compile & Test

Rule is uploaded to the Docker sandbox. YARA CLI compiles it and runs it against the actual binary. Result: VALID_MATCH, VALID_NO_MATCH, or INVALID_RUN.

YARA CLILive Sandbox
7

Stateful Repair Loop

On failure, the exact error is fed back into the same chat session. Compile errors inject violation lists. VALID_NO_MATCH injects per-string match telemetry. Up to 6 cycles before final result.

Strict Architectural Constraints

Two generation paths. Zero tolerance for policy violations.

The policy gate runs deterministically before and after every AI response. Violations send the rule back for correction automatically.

🔵 Path A — PE Binary Rules

  • Minimum 7 unique high-signal strings
  • MUST include pe.imphash() — locks import table behavior
  • MUST include pe.version_info — locks version metadata
  • Hybrid OR condition — string-only fallback fires even if PE headers are wiped in memory
  • Condition threshold: exactly total_strings - 2
  • import "pe" auto-injected if missing

🟦 Path B — Non-PE (Scripts, PowerShell, Docs)

  • Minimum 15 unique high-signal strings (7 if low-signal sample)
  • STRICT BAN on pe.* module usage
  • Condition threshold: exactly total_strings - 2
  • Relaxed string scoring — PS variables and .NET type casts receive non-zero scores
  • String cap raised to 120 to give the AI enough raw material
  • No filesize constraints — rule must work in memory
🔒 Universal hard blocks (both paths): Regex string definitions (/pattern/), pe.sections[i].entropy, bare imphash identifier without pe., and any filesize constraint.

Sample Output

A real rule. Validated. Retrohunt-ready.

Generated in one pass, tested against the actual binary, zero manual editing required.

Win64_Stealer_Crypt32_e25892.yar
✓ VALID_MATCH
import "pe"

rule Win64_Stealer_Crypt32_e25892603
{
  meta:
    author = "YARA-GPT"
    description = "Detects credential-stealing PE with crypt32 certificate store manipulation"
    sha256 = "e25892603c42e34bd7ba0d8ea73be600d898cadc..."

  strings:
    $s1 = "CertDuplicateCertificateChain" ascii wide
    $s2 = "CertAddCertificateContextToStore" ascii wide
    $s3 = "WakeByAddressSingle" ascii wide
    $s4 = "ProcessPrng" ascii wide
    $s5 = "CertEnumCertificatesInStore" ascii wide
    $s6 = "CertGetCertificateChain" ascii wide
    $s7 = "CertFreeCertificateChain" ascii wide

  condition:
    // Hybrid OR — PE anchors for disk, string-only fallback for memory dumps
    (pe.imphash() == "a1b2c3d4e5f6..."
     and pe.version_info["ProductName"] == "CertMgr Utility"
     and 5 of ($s*))
    or 5 of ($s*)
}
