The Problem

Writing YARA rules is hard.
Doing it at scale is harder.

Security teams waste hours crafting rules manually — only to find they miss the sample or flood the pipeline with false positives.

⏱️

Hours per rule

Manual YARA authoring requires deep reverse engineering knowledge. A single quality rule can take hours of binary analysis.

🎯

Rules that miss the sample

Without validating against the actual binary, rules often miss the target — especially in memory-scanning scenarios where PE headers are wiped.

🔥

False positive floods

Generic strings produce rules that match everything. Analysts need rare, sample-specific indicators — which takes expertise to identify.

📋

No unified workflow

Pivoting between VirusTotal, IDA, CAPA, and a text editor for every hash is fragmented and not repeatable at scale.

How It Works

A fully automated pipeline,
end to end.

Paste a hash. The rest is handled automatically.

1

Intelligence Gathering

The system queries VirusTotal for the file report, threat labels, tags, and detection ratio. Optionally enriches with AlienVault OTX and OpenCTI for campaign and actor attribution.

VirusTotal API OTX OpenCTI
2

Sandbox Analysis

The actual sample is downloaded and injected into a hardened Docker container. analyze.py extracts strings, parses PE metadata (imphash, imports, exports, sections), measures entropy, and runs CAPA behavioral analysis.

Docker pefile CAPA strings
3

AI Rule Generation

All artifacts are fed into a Google Gemini stateful chat session. The AI generates a YARA rule following strict quality constraints. If the rule fails validation, the exact error is fed back into the same session — up to 6 correction cycles.

Gemini 2.5 Flash Stateful Chat YARA CLI

Features

Built for the way analysts actually work.

⚡

One-Click YARA Generation

Paste any MD5, SHA1, or SHA256. The full pipeline runs automatically and returns a validated, ready-to-deploy rule.

🔄

Stateful AI Repair Loop

Up to 6 correction cycles. The AI sees its own errors and corrects them in the same conversation — no context loss between retries.

🧬

Dual-Path Logic

PE binaries get structural anchors (pe.imphash + pe.version_info) with hybrid OR for memory scanning. Scripts/PowerShell/JS require 15 strings to prevent false positives.

🎯

Smart No-Match Feedback

If the rule compiles but doesn't match the sample, the system tells the AI exactly which string identifiers matched and which didn't — and asks it to fix accordingly.

📦

Bulk Processing

Upload a list of hundreds of hashes. Process them sequentially. Download a ZIP of all rules or a single Retrohunt-ready file of only validated rules.

🔍

VirusTotal Retrohunt Export

One button. Clean concatenation of valid rules with deduplicated imports — ready to upload directly to VT Retrohunt without any manual editing.

🛡️

Docker Sandbox Isolation

All sample analysis happens in a hardened container: 1GB memory limit, no-new-privileges, isolated from the host filesystem.

🧠

AI Expert Report

A separate Gemini call produces structured threat attribution: malware family, campaign, MITRE ATT&CK mapping, confidence score, and recommended response actions.

🌐

Full CTI Enrichment

VirusTotal detections, AlienVault OTX pulses, and OpenCTI structured intelligence — all in one view per hash.

🗄️

Searchable History Database

Every analysis is stored in SQLite. Filter by hash, date, or YARA result. Export any entry as a full PDF report.

Architecture

How the pipeline is wired.

A single hash triggers a coordinated sequence across four distinct systems.

Hash Input MD5 / SHA1 / SHA256
→
VirusTotal API Report + Download
→
Docker Sandbox analyze.py · CAPA · PE
→
Google Gemini Stateful Chat · 6 cycles
→
YARA CLI Compile + Match Test
→
Result VALID_MATCH / Export
🔬

analyze.py (in container)

Extracts strings (ASCII + UTF-16LE), parses PE headers with pefile, measures Shannon entropy, and runs CAPA with a 180-second timeout. Outputs structured artifacts.json.

🤖

Gemini Chat Session

One persistent session per hash. Temperature 0.2, max 4096 tokens. Policy violations and compile errors are fed back into the same session to preserve full context across repair cycles.

📏

Policy Gate

Deterministic checks before and after every AI response: string count, condition threshold (N−2), PE anchor presence, regex ban, entropy field ban, import statement validation.

import "pe"

rule Win64_Stealer_Crypt32_e25892603
{
  meta:
    author = "YARA-GPT"
    description = "Detects credential-stealing PE with crypt32 certificate store manipulation"
    sha256 = "e25892603c42e34bd7ba0d8ea73be600d898cadc..."
    date = "2026-03-22"

  strings:
    $s1 = "CertDuplicateCertificateChain" ascii wide
    $s2 = "CertAddCertificateContextToStore" ascii wide
    $s3 = "WakeByAddressSingle" ascii wide
    $s4 = "ProcessPrng" ascii wide
    $s5 = "CertEnumCertificatesInStore" ascii wide
    $s6 = "CertGetCertificateChain" ascii wide
    $s7 = "CertFreeCertificateChain" ascii wide

  condition:
    // Hybrid OR: PE anchors for disk scanning, string-only fallback for memory dumps
    (pe.imphash() == "a1b2c3d4e5f6..."
     and pe.version_info["ProductName"] == "CertMgr Utility"
     and 5 of ($s*))
    or 5 of ($s*)
}
